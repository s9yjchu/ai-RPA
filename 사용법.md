## 3. 수동 실행 방법

자동 실행이 실패했거나 특정 날짜를 다시 업데이트해야 할 때 사용합니다.

### 방법 1: 바탕화면 바로가기 더블클릭 (최종 사용자)

설치 시 바탕화면에 생성된 **"B2C 고객지표 수동실행"** 아이콘을 더블클릭하면 즉시 실행됩니다.

실행 완료 시 결과 창이 표시됩니다:
- **[완료]** → Google Sheets 에서 결과 확인 후 창 닫기
- **[오류]** → `logs\` 폴더 로그 파일을 관리자에게 전달

### 방법 2: run_rpa.bat 더블클릭 (바로가기 없는 경우)

프로젝트 폴더의 `run_rpa.bat` 를 더블클릭하면 **어제 날짜**를 기준으로 실행됩니다.

### 방법 3: 특정 날짜 지정 실행

PowerShell 또는 명령 프롬프트에서:

```powershell
cd C:\Users\S9_User\projects\ai-RPA
.venv\Scripts\python -m src.main daily --date 2026-06-01 --force
```

- `--date YYYY-MM-DD` : 업데이트할 날짜 지정
- `--force` : 이미 성공 처리된 날도 강제 재실행

### 방법 3: 설정 마법사의 [지금 실행] 버튼

`setup.bat` 실행 → 설정 마법사 창 → **[지금 실행]** 버튼 클릭

## 4. 로그 확인 방법

실행 로그는 `logs/` 폴더에 자동 저장됩니다.

```
logs/
├── 20260602_083012.log    ← 실행 시각별 로그 파일
├── 20260602_090015.log
└── debug/
    ├── 20260602_083012_hub_02_after_login.png   ← 단계별 스크린샷
    ├── 20260602_083012_hub_04_sis_menu.png
    └── ...
```

### 로그 파일 열기

```powershell
# 가장 최근 로그 열기
Get-ChildItem logs\*.log | Sort-Object LastWriteTime | Select-Object -Last 1 | notepad
```

또는 `logs/` 폴더에서 가장 최근 `.log` 파일을 메모장으로 엽니다.

### 성공 여부 확인 키워드

| 키워드 | 의미 |
|---|---|
| `[DONE] YYYY-MM-DD 업데이트 완료` | 완전 성공 |
| `[DRY_RUN]` | 테스트 모드 — 실제 입력 안 됨 |
| `DataNotReadyError` | 데이터 미준비 — 재시도 예정 |
| `[FATAL]` | 오류 발생 — 스택 트레이스 확인 필요 |

## 5. 실행 상태 파일

RPA 는 중복 실행을 방지하기 위해 `state/` 폴더에 JSON 파일을 저장합니다.

```
state/
├── daily_2026-06-01.json   ← 일별 실행 결과
└── monthly_2026-06.json    ← 월별 실행 결과
```

JSON 파일을 열면 아래와 같은 내용이 있습니다:

```json
{
  "status": "success",       ← success / failed / pending
  "attempts": 1,             ← 총 시도 횟수
  "completed_at": "2026-06-01T08:35:12+09:00"
}
```

**특정 날짜를 초기화(재실행 허용)하려면** 해당 JSON 파일을 삭제하고 `--force` 옵션으로 실행합니다.

```powershell
Remove-Item state\daily_2026-06-01.json
.venv\Scripts\python -m src.main daily --date 2026-06-01 --force
```

## 6. 자주 있는 문제와 해결 방법

### 문제 1: 오전에 "데이터 미준비" 이메일이 계속 옴

**원인**: OLAP 에 전날 데이터가 아직 반영되지 않은 상태입니다.

**해결**: 30분마다 자동 재시도하므로 보통 10:00~11:00 경 자동 처리됩니다.
11:30 이후에도 해결되지 않으면 OLAP 담당자에게 문의하세요.

---

### 문제 2: "업데이트 실패" 이메일이 옴

**원인**: 네트워크 오류, OLAP 시스템 점검, 비밀번호 변경 등 다양한 원인이 있습니다.

**해결 순서**:

1. `logs/` 에서 최근 `.log` 파일을 열어 `[FATAL]` 이 포함된 줄을 확인합니다.
2. **비밀번호 변경**이 원인인 경우:
   `setup_user.bat` 을 관리자 권한으로 다시 실행하여 새 비밀번호를 입력합니다.
3. **네트워크/VPN** 문제인 경우: VPN 연결 상태를 확인합니다.
4. 해결 후 수동 실행:
   ```powershell
   .venv\Scripts\python -m src.main daily --force
   ```

---

### 문제 3: Sheets 에 입력이 안 됨 (로그는 성공)

**원인 1**: `DRY_RUN=1` 로 설정된 경우 (테스트 모드).
→ `.env` 파일에서 `DRY_RUN=0` 으로 변경 후 재실행합니다.

**원인 2**: Google 인증 토큰(`token.json`) 만료.
→ `token.json` 을 삭제하고 재실행하면 브라우저에서 Google 로그인 화면이 열립니다.
```powershell
Remove-Item token.json
.venv\Scripts\python -m src.main daily --force
```

---

### 문제 4: 특정 날짜가 입력되지 않음

**원인**: 해당 날짜의 OLAP 데이터가 존재하지 않거나, 이미 `success` 상태로 기록된 경우.

**해결**:
```powershell
# 상태 초기화 후 강제 재실행
Remove-Item state\daily_2026-06-01.json
.venv\Scripts\python -m src.main daily --date 2026-06-01 --force
```

---

### 문제 5: GCP 에이전트 서비스가 자동 실행이 안 됨

**확인 방법**:
```bat
nssm status B2C_RPA_Agent
```
또는: 시작 메뉴 → "서비스" 검색 → `B2C RPA GCP Agent` 찾기

**재시작 방법 (관리자 권한)**:
```bat
nssm restart B2C_RPA_Agent
```

**재설치**:
`install_agent_service.bat` 을 관리자 권한으로 실행

## 7. GCP 에이전트 운영

자동 실행은 **GCP Cloud Scheduler → Pub/Sub → Windows 에이전트** 방식으로 동작합니다.

### 에이전트 서비스 관리

| 명령 | 동작 |
|---|---|
| `nssm status B2C_RPA_Agent` | 서비스 상태 확인 |
| `nssm start B2C_RPA_Agent` | 시작 |
| `nssm stop B2C_RPA_Agent` | 중지 |
| `nssm restart B2C_RPA_Agent` | 재시작 (`.env` 변경 후 필요) |

에이전트 로그: `logs\agent_stdout.log`, `logs\agent_stderr.log`

### GCP Cloud Scheduler 스케줄

| 작업 | 시간 (KST) | 발화 횟수 | 설명 |
|---|---|---|---|
| `daily-rpa-*` | 평일 08:30~11:30 | 30분 간격 × 7회 | 일별 업데이트 |
| `monthly-rpa-*` | 매월 1~4일 08:30~11:30 | 30분 간격 × 28회 | 월별 업데이트 |

GCP 콘솔 확인: Cloud Scheduler → `ai-rpa-daily-*` / `ai-rpa-monthly-*`

### 수동 테스트 (GCP 없이)

```powershell
run_agent.bat --test daily    # 에이전트를 거치지 않고 daily 직접 실행
run_agent.bat --test monthly  # monthly 직접 실행
```

## 8. 설정 변경 방법

### SPC Hub 비밀번호가 변경된 경우

`setup_user.bat` 을 관리자 권한으로 다시 실행하세요.  
이미 설치된 환경이라면 패키지/브라우저 설치는 건너뛰고 자격증명만 덮어씁니다.

### 직접 수정 (개발자용)

프로젝트 폴더의 `.env` 파일을 메모장으로 열어 수정합니다:

| 항목 | 환경변수 |
|---|---|
| SPC Hub 아이디 | `SPCHUB_ID` |
| SPC Hub 비밀번호 | `SPCHUB_PW` |
| 알림 발송 이메일 | `REPORT_SENDER` |
| 알림 수신 이메일 | `REPORT_RECIPIENTS` (쉼표로 구분) |

### Google 인증 갱신

`token.json` 을 삭제하고 `setup_user.bat` 을 다시 실행하면 브라우저에서 Google 로그인 화면이 열립니다.

```powershell
Remove-Item token.json
# 이후 setup_user.bat 실행
```

## 9. 월별 업데이트 (매월 1일)

매월 1일 아침에 자동으로 전월 데이터를 업데이트합니다.

- **LOG REPORT** → 해피앱 월 로그인객수 (Sheets O열)
- **VISUAL REPORT** → 해피앱 MAU (Sheets P열)

> **주의**: 월별 데이터는 전월 마감 후 반영되므로 2~3일 지연될 수 있습니다.
> 매월 1일~4일 중 데이터가 준비되면 자동 처리됩니다.

수동 실행:
```powershell
.venv\Scripts\python -m src.main monthly --year 2026 --month 5 --force
```
