# B2C 고객지표 RPA 설치 및 사용 가이드

> 이 문서는 **처음 설치하는 담당자**를 위한 안내입니다.

---

## 1. 준비사항

설치 전 아래 항목을 확인하세요.

| 항목 | 확인 내용 |
|---|---|
| PC | **사내 네트워크**에 연결된 Windows PC (VPN 또는 사무실 내 PC) |
| Python | **별도 설치 불필요** — 설치 프로그램이 자동으로 처리합니다 |
| credentials.json | 관리자에게 별도로 받은 파일 — **이 폴더에 미리 복사해두어야 합니다** |
| 파일 | 이 폴더 안의 모든 파일이 있는지 확인 |

> ⚠️ **주의**: `credentials.json` 파일이 없으면 설치가 진행되지 않습니다. 설치 전 관리자에게 요청하세요.

## 2. 최초 설치 (한 번만 실행)

### 방법: `setup_user.bat` 더블클릭

1. 이 폴더에서 **`setup_user.bat`** 을 마우스 오른쪽 클릭 → **관리자 권한으로 실행**
2. 화면의 안내에 따라 진행합니다:

   **① Python 자동 설치** (필요한 경우)
   ```
   [설치] Python 3.12 설치 중 (약 1~2분 소요)...
   ```
   PC에 Python이 없거나 호환되지 않는 버전이면 자동으로 설치됩니다. 기다려주세요.

   **② 패키지 및 브라우저 설치**
   ```
   [설치] 필요 패키지 설치 중 (최초 실행 시 3~5분 소요)...
   [설치] Chromium 브라우저 설치 중 (최초 실행 시 2~3분 소요)...
   ```

   **③ SPC Hub 로그인 정보 입력**
   ```
   SPC Hub 아이디 : (본인 사내 아이디 입력 — 예: jihye.jeon)
   SPC Hub 비밀번호: (본인 사내 비밀번호 입력, 화면에 표시 안 됨)
   ```

   **④ Google 계정 인증**
   ```
   Google 이메일 주소: (예: jihye.jeon@spc.co.kr)
   ```
   > SPC Hub 아이디와 **다를 수 있습니다**. B2C사업본부 고객지표 스프레드시트의 **편집 권한이 있는 Google 계정**을 입력하세요.

   입력 후 브라우저가 자동으로 열립니다. 해당 Google 계정으로 로그인하고 **[허용]** 을 클릭하세요.

3. "설치 완료" 메시지가 나오면 완료입니다.
4. **바탕화면에 "B2C 고객지표 수동실행" 바로가기가 자동 생성됩니다.**

## 3. 자동 실행 확인

설치 후 RPA 는 **매일 오전 09:00** 부터 자동으로 실행됩니다.

- **PC 가 켜져 있어야 합니다** (절전 모드는 괜찮습니다)
- **사내 네트워크 연결 상태**여야 합니다

### 실행 시간표

| 시도 | 시간 |
|---|---|
| 1차 시도 | 09:00 |
| 2차 시도 | 09:30 (데이터 미준비 시) |
| 3차 시도 | 10:00 |
| 4차 시도 | 10:30 |
| 5차 시도 | 11:00 |
| 6차 시도 | 11:30 |
| 최종 시도 | 12:00 |

데이터가 준비되면 중간에 자동 완료됩니다. 이미 완료된 날은 건너뜁니다.

자동 실행 시 아래와 같은 창이 화면에 나타납니다. 확인 후 닫으셔도 됩니다.

**성공 시:**
```
============================================================
  [완료] 정상적으로 실행되었습니다.

  Google Sheets "B2C사업본부 고객지표" 에서 결과를 확인하세요.
  확인 후 이 창을 닫으셔도 됩니다.
============================================================
```

**실패 시:**
```
============================================================
  [오류] 실행 중 문제가 발생했습니다.

  logs\ 폴더의 최근 .log 파일을 관리자에게 전달해주세요.
  담당: andrew.chu@secta9ine.co.kr
============================================================
```

실행 완료 후 **Google Sheets "B2C사업본부 고객지표"** 에서 확인합니다.

## 5. 수동 실행

자동 실행이 실패했거나 직접 실행하고 싶을 때 사용합니다.

### 방법: 바탕화면 "B2C 고객지표 수동실행" 더블클릭

설치 후 바탕화면에 생성된 바로가기를 더블클릭하면 즉시 실행됩니다.

실행이 완료되면 아래와 같은 결과 창이 표시됩니다:
- **[완료]** 메시지 → Google Sheets 에서 업데이트 결과 확인 → 창 닫기
- **[오류]** 메시지 → `logs\` 폴더의 로그 파일을 관리자에게 전달

> 바로가기가 없는 경우 RPA 폴더 안의 `run_rpa.bat` 을 더블클릭해도 됩니다.

## 6. 문제 해결

### ① `'wright'` 또는 `'Read-Host'` 오류가 납니다

**원인**: `.bat` 파일의 줄바꿈 문자가 잘못되어 있습니다.
**해결**: 관리자에게 최신 `B2C_고객지표_RPA.zip` 파일을 재전달 받아 다시 압축 해제 후 실행하세요.

### ② Python 설치 오류가 납니다

**원인**: 인터넷 연결이 불안정하거나, 이미 호환되지 않는 Python이 설치된 경우입니다.
**해결**:
1. 인터넷 연결을 확인하세요.
2. 그래도 안 된다면 기존 폴더를 삭제하고 새로 압축을 해제한 뒤 다시 실행하세요.
3. 관리자에게 문의하세요.

### ②-1 실행 시 `ModuleNotFoundError` (예: `No module named 'gspread'`)

**원인**: 설치 후 **폴더를 옮기거나 이름을 바꾼** 경우입니다. 설치 폴더(`.venv`)는
원래 위치에 고정되어 있어, 옮기면 내부 연결이 끊겨 패키지를 찾지 못합니다.
**해결**: 폴더를 옮기지 말고, **새 위치에 zip 을 새로 압축 해제한 뒤** `setup_user.bat`
을 다시 실행하세요. (옛 폴더의 내용을 복사해 옮기지 마세요.)

### ③ "SPC 로그인이 안 됩니다"

- 실패 메일/로그에 **`SPC Hub 로그인 실패 — SPCHUB_ID/SPCHUB_PW 를 확인하세요`** 가
  보이면 아이디·비밀번호 문제입니다. (SPC 비밀번호는 주기적으로 만료/변경됩니다.)
- 먼저 브라우저에서 `hub.spc.co.kr` 에 해당 ID/PW 로 **직접 로그인** 되는지 확인하세요.
  안 되면 SPC 헬프데스크에서 비밀번호 재설정/잠금 해제 후 진행.
- 비밀번호가 변경된 경우: `setup_user.bat` 을 다시 실행하여 새 비밀번호 입력.
  설정 마법사가 **저장 직후 실제 로그인으로 검증**하므로, ✅ 가 떠야 올바른 값입니다.
- VPN 연결 상태 확인 (사무실 외부인 경우)

### ④ "Google 인증이 안 됩니다"

- `token.json` 파일을 삭제하고 `setup_user.bat` 을 다시 실행
- B2C사업본부 고객지표 시트의 **편집 권한이 있는** Google 계정으로 로그인했는지 확인

### ⑤ "데이터가 입력되지 않았습니다"

- `logs\` 폴더의 최근 `.log` 파일을 열어 오류 내용 확인
- 개발 담당자에게 로그 파일 공유

### ⑥ "작업 스케줄러에서 찾을 수 없습니다" / 자동 실행이 안 됩니다

- `setup_user.bat` 을 관리자 권한으로 다시 실행
- 작업 스케줄러 확인: `Win + R` → `taskschd.msc` → 작업 스케줄러 라이브러리 에서 `B2C_RPA_Daily_*` 확인

## 7. 로그 파일 위치

실행 기록은 `logs\` 폴더에 날짜별로 저장됩니다.

```
logs\
  20260616_090012.log    ← 09:00에 실행된 로그
  20260616_093015.log    ← 09:30 재시도 로그
```

로그 파일을 메모장으로 열어 맨 마지막 줄을 확인합니다:
- `[DONE] ... 업데이트 완료` → 성공
- `ERROR` 포함 → 실패, 파일 전체를 관리자에게 전달

> 💡 **실패 시 로그·화면 캡처가 실패 알림 메일에 자동 첨부**됩니다.
> 대부분의 경우 따로 파일을 보내지 않아도 관리자가 원인을 바로 확인할 수 있습니다.
> (관리자 옵션) `.env` 에 `LOG_UPLOAD_DRIVE=1` 설정 시 구글 드라이브 공유 폴더에도
> 자동 업로드됩니다. 이 옵션을 켠 뒤에는 `token.json` 을 삭제하고 `setup_user.bat`
> 을 다시 실행해 **드라이브 권한을 추가로 허용**해야 합니다.

문제가 있을 때 이 파일을 개발 담당자에게 보내주세요.
